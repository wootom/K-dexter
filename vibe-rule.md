# 🛡️ Project Vibe & Rules: Plan Mode First

이 파일은 프로젝트의 최상위 가이드라인이며, 모든 작업은 본 문서의 절차를 준수해야 합니다.

---

## 1. 🔍 기본 작동 원칙 (The Mindset)
- **Plan First, Code Later:** 즉각적인 코드 생성보다 문제의 본질 파악과 설계가 우선입니다.
- **Context Awareness:** 프로젝트 내의 다른 파일들(`PRD`, `README`, 기존 코드)과의 의존성을 먼저 탐색합니다.
- **Safety Over Speed:** 파괴적인 변경(Destructive change)을 피하고, 기존 로직과의 호환성을 최우선으로 합니다.

---

## 2. 📝 Plan Mode 절차 (Essential)
모든 요청에 대해 코드를 작성하기 전, 반드시 다음 단계에 따라 **'작업 계획'**을 먼저 브리핑합니다.

### Phase 1: 탐색 및 분석 (Investigation)
- 요청받은 기능과 관련된 기존 파일 및 로직을 탐색합니다.
- 모호한 요구사항이나 기획 의도와 충돌하는 부분이 있다면 즉시 질문합니다.

### Phase 2: 설계 브리핑 (Design Brief)
사용자에게 아래 내용을 포함한 계획을 먼저 제시하고 승인을 기다립니다:
1. **수정/신규 생성 파일 리스트:** (`ex: src/components/StockChart.tsx`)
2. **핵심 로직 변경 요약:** 어떤 로직이 추가되거나 바뀌는지 설명.
3. **사이드 이펙트 체크:** 이 작업으로 인해 영향받을 수 있는 다른 기능들.
4. **성공 기준 (Testing):** 작업이 완료되었음을 어떻게 검증할 것인가?

### Phase 3: 실행 및 업데이트 (Execution)
- 승인된 계획에 따라서만 코드를 작성합니다.
- 작업 완료 후, 변경 사항이 프로젝트의 다른 `.md` 가이드라인에도 반영되어야 한다면 제안합니다.

---

## 3. 🛠️ 기술 및 스타일 가이드 (Coding Standards)
- **Clean Logic:** 기획자가 읽어도 이해할 수 있는 명확한 변수명과 함수 구조를 유지합니다.
- **Error Handling:** 예외 상황(Exception)에 대한 처리를 반드시 포함합니다.
- **No URLs in Reports:** 생성되는 리포트나 주석에 불필요한 URL 링크를 포함하지 않습니다. (텍스트 중심의 설명 지향)

---

## 4. 🧠 장기 기억 관리 (Memory Bank)
- 주요 기획적 의사결정이나 기술적 해결책은 세션이 끝나기 전 요약하여 제공합니다.
- 사용자가 요청할 경우, 프로젝트의 `INSTRUCTIONS.md`나 `DECISION_LOG.md`를 업데이트하여 맥락을 영구 보존합니다.
